## 説明 / Overview
　このアドオンは、記入した数式の通りに数値計算をする拡張mathノード「**Math Controller**」を追加します。  
Math Controllerは内部に数式ノード群を生成するカスタムグループノードです。  
ノードはシェーダー・ジオメトリー・コンポジター・テクスチャノードエディターで使えます。(※テクスチャノードはサブグループノードにしかソケットが入りません。)  
  This addon add **Math Controller**. This is Advanced math node that acts acoording to fromulas your Written.
Math Controller is CustomGroupNode that generates math nodes inside.
You can use thsi node in Shader Editer, Geometry Node Editor, Compositor and Texture node editor.
(※In Texture node editor,only Sub Groups of Math cntrollers have sokets links. )

## 使い方 / Usage
### 場所 / location
#### blender > 5.0
- shader & geometry & compositor  
![img1](<https://github.com/o-oo-gote510/Math-Controller-Addon/blob/main/images/img1.jpg?raw=true>)  
- texture node editor
Node editor > Add > Converter > "Math Controller"
#### 4.2 <= blender < 5.0
- shader & geometry  
Node editor > Add > Utilities > "Math Controller"
- texture & compositor  
Node editor > Add > Converter > "Math Controller"
### `+`/`-` ボタン
入力フォームを増減。
### 関数入力フォームについて
実数、変数、[演算子・関数一覧] にある関数、および他のフォームの出力（`f1`, `f2` など）を組み合わせて数式を記述できます。
- 変数ソケットの追加: 数式内で使用した変数は、自動的に入力ソケットとして追加されます。変数が使用されなくなり、かつ入力接続がない状態になると、そのソケットは自動的に削除されます。
- 結果の継承: 各入力フォームには関数名（例: `f1`, `f2`）が割り当てられており、これを他の入力欄で変数として使用することで、計算結果を継承できます。
### 拡張数式エディタボタン (Advanced Formula Editor)
入力フォームの横にあるボタンから拡張エディタが開きます。
- 長い数式の編集、括弧 () の階層構造の可視化、および関数一覧の確認が可能です。
- 関数の引数: 各関数の引数は、既存のMathノードの入力ソケットに昇順で対応しています。
### ノードグループの生成ボタン
フォーム下部のボタンから、同じノード空間を共有してMath Controllerと同期している、通常のノードグループを作成できます。
- 内部の編集: このノードグループから、Math Controller内部のノード群にアクセス・編集することができます。
- 注意: Math Controllerは更新時にグループ内部の構造を上書きします。手動で加えた編集を維持したい場合は、Math Controllerノード自体を削除するか、リンクを切断することを推奨します。
また、Math Controllerがある状態で内部を編集すると、その際に更新サイクルから外れたノード群が内部に残る可能性があるため注意してください。
### 複製
標準のノードと同様に、`Shift + D`や `Ctrl + Shift + D`により別ノードとして複製されます。

### `+`/`-` Buttons
Add or remove input fields.

### Formula Input Fields
You can write formulas by combining real numbers, variables, functions from the [Operators & Functions List], and outputs from other fields (e.g., `f1`, `f2`).
- **Dynamic Variable Sockets**: Variables used in your formula are automatically added as input sockets. If a variable is no longer used and has no incoming connection, its socket will be automatically removed.
- **Result Inheritance**: Each input field is assigned a function name (e.g., `f1`, `f2`). You can use these as variables in other fields to inherit calculation results.

### Advanced Formula Editor Button
Click the button next to the input field to open the Advanced Formula Editor.
- **Features**: Edit long formulas comfortably, visualize parenthesis `()` nesting levels, and browse the function list.
- **Function Arguments**: Function arguments correspond to the standard Math node input sockets in ascending order.

### Generate Node Group Button
The button at the bottom creates a standard Node Group that shares the same node space and stays synchronized with the Math Controller.
- **Internal Editing**: You can access and edit the internal node network generated by the Math Controller through this Node Group.
- **Caution**: The Math Controller overwrites the internal structure of the group during updates. If you wish to maintain manual changes, we recommend deleting the Math Controller node or breaking the link. 
Additionally, please note that modifying the internal network while the Math Controller is active may leave "orphaned" nodes that are no longer part of the update cycle.

### Duplication
Like standard Blender nodes, the Math Controller can be duplicated as a separate node using `Shift + D` or `Ctrl + Shift + D`.
